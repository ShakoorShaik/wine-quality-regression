3. Experiment design and model sweep¶

This section fixes one shortlist of ANN architectures and then evaluates that same shortlist under three major activation-function batches: ReLU, tanh, and sigmoid.

3.1 Shared architecture family and hyperparameter ranges¶

Every batch below uses the same 8 ANN architectures from utils.py. The input layer always has 12 nodes because the wine dataset has 12 features after preprocessing, and the output layer always has 1 linear node because this is a regression problem.

Selected architectures and why they are included:

  • 12 -> 8 -> 1: smallest nonlinear model; checks whether a very compact network is already enough.
  • 12 -> 16 -> 1: strong default shallow network; adds moderate capacity without becoming large.
  • 12 -> 32 -> 1: tests whether a wider shallow model improves over 8 or 16 units.
  • 12 -> 64 -> 1: upper bound for shallow width; useful for checking whether more width actually helps.
  • 12 -> 16 -> 8 -> 1: smallest tapered deep model; tests whether depth helps beyond a small 1-layer ANN.
  • 12 -> 32 -> 16 -> 1: strongest medium-sized tapered 2-layer candidate; balances expressiveness and control.
  • 12 -> 64 -> 16 -> 1: tests a wide first hidden layer followed by compression into a bottleneck.
  • 12 -> 64 -> 32 -> 1: strongest larger 2-layer candidate worth keeping before the model class becomes unnecessarily large.

Why the four hidden-layer sizes 8, 16, 32, and 64 were chosen:

  • they provide a clean progression from small to moderately wide networks
  • they are enough to test under-capacity, reasonable capacity, and mild over-capacity on a 12-feature dataset
  • they keep the search interpretable and computationally manageable

Why smaller sizes were not emphasized:

  • widths below 8 would likely be too restrictive for modeling nonlinear structure in this dataset
  • very small widths are less informative because they behave more like severe bottlenecks than realistic ANN candidates

Why larger sizes were not included:

  • the dataset has only 12 input features, so widths above 64 are hard to justify
  • the current ANN training code uses plain batch gradient descent without early stopping or more advanced optimization, so very large models are more likely to add instability and runtime than value

Learning-rate values tested: 1e-4, 5e-4, 1e-3, 5e-3

  • 1e-4 checks whether training needs a conservative step size for stable optimization
  • 5e-4 and 1e-3 cover reasonable middle-ground values for this scale of tabular regression problem
  • 5e-3 tests whether the network benefits from more aggressive optimization or becomes unstable

L2 values tested: 0, 1e-5, 1e-4, 1e-3

  • 0 gives an unregularized baseline
  • 1e-5 tests very light regularization
  • 1e-4 is a moderate default regularization strength
  • 1e-3 checks whether stronger shrinkage improves generalization for wider models

Epoch values tested: 2000 and 5000

  • 2000 is a moderate training budget
  • 5000 checks whether additional training materially improves the fit

def run_ann_experiment(batch, hidden_layers, learning_rate, l2_coeff, max_epochs):
    architecture_label = format_architecture(hidden_layers)
    model = ANNRegression(
        num_layers=len(hidden_layers),
        num_units=hidden_layers,
        activation=batch["activation"],
        learning_rate=learning_rate,
        max_epochs=max_epochs,
        batch_size=batch_size,
        l2_coeff=l2_coeff,
        random_state=42,
    )

    if l2_coeff > 0:
        model.fit_with_l2_regularization(normed_X_train, y_train_smooth)
    else:
        model.fit(normed_X_train, y_train_smooth)

    test_pred = model.predict(normed_X_test).ravel()
    mse, rmse, mae, r2, acc1 = cm(y_test, test_pred)

    return {
        "subsection": batch["subsection"],
        "batch_name": batch["batch_name"],
        "description": batch["description"],
        "activation": batch["activation"],
        "learning_rate": learning_rate,
        "max_epochs": max_epochs,
        "batch_size": batch_size,
        "l2_coeff": l2_coeff,
        "num_hidden_layers": len(hidden_layers),
        "hidden_layers": list(hidden_layers),
        "architecture": architecture_label,
        "mse": mse,
        "rmse": rmse,
        "mae": mae,
        "r2": r2,
        "acc1": acc1,
        "loss_history": model.loss_history,
        "model": model,
        "y_pred": test_pred,
    }

def regularized_loss(model, X, y, l2_coeff):
    pred = model.predict(X).reshape(-1, 1)
    mse = float(np.mean((pred - y) ** 2))
    l2_penalty = 0.0
    if l2_coeff > 0:
        l2_penalty = sum(float(np.sum(coef ** 2)) for coef in model.coefs_)
    return mse + l2_coeff * l2_penalty
